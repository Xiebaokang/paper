\cabstract{
% 随着Transformer架构在自然语言处理、代码生成和大模型推理等任务中的广泛应用，其中矩阵乘法（Matrix Multiplication, Matmul）和多头注意力机制（Multi-Head Attention, MHA）的执行效率已成为影响模型部署性能的关键因素。由于异构硬件平台在模型执行、内存层次结构和软件生态系统方面存在显著差异，依赖供应商高性能库或手动编写优化内核的传统方法面临着适应性复杂、开发成本高昂以及性能可移植性不足等挑战。与此同时，现有的深度学习编译器在块级逻辑表达、低级内存访问控制以及跨平台性能可移植性方面仍存在一定局限，难以满足 Transformer 关键算子高性能自动代码生成的需求。为解决Transformer模型关键算子自动生成中的上述问题，本文提出 DeepGen，这是一个基于 MLIR 的高性能算子生成框架。本文的主要贡献如下：
% (1) 我们提出了一种专为 Transformer 关键算子量身定制的块级分层中间表示（IR）方法。为解决现有编译器面临的挑战——即在代码生成过程中难以保留块级语义，以及无法显式表达计算和内存访问结构——我们设计了基于 MLIR（Multi-Level Intermediate Representation） 的 DeepGen 方言（Dialect）。该方言为矩阵乘法和多头注意力机制中的块级计算逻辑、数据访存以及并行映射关系提供了一个统一的模型。通过显式定义 Alloc Buffer、Copy、Matmul 和 Block 等操作，该方法将计算逻辑与数据访存解耦，为后续优化和跨平台性能移植提供了统一且清晰的表示基础。
% (2) 我们提出了一种基于推断 IR 降级（Lowering）信息的自动降级机制。为了解决从块级抽象映射到低级线程级执行逻辑的难题，本文构建了一种专注于数据依赖关系的索引推断方法。该方法能够根据算子的计算模式，自动推导出线程块和线程束内的索引布局，从而实现从块级表示到线程级表示的渐进式转换。该方法有效确保了算子语义在不同抽象层级间的正确传递，并为后续的内存访问优化、并行调度及低级代码生成奠定了基础。
% (3) 我们提出了一套专为异构平台定制的跨平台优化与自动调优方法。为解决不同硬件平台在内存访问结构和执行特性方面的差异，我们设计了软件流水线（Software Pipeline）、局部规约轴切分（Local Split-K）、线程块访存重排（Block Swizzle）和离散化访存（Access Scatter）等优化 Pass。这些优化 Pass 从多个层面提升了生成的代码的执行效率，包括计算调度、规约拆分、线程块映射和共享存储访问。此外，为应对庞大的参数空间及手动调优成本高昂的挑战，我们构建了一个基于空间剪枝的自动调优框架。该框架通过生成候选参数空间并剔除无效配置，自动搜索针对目标硬件的优化执行配置，从而提高了算子生成的自动化程度。
% (4) 在 NVIDIA A100/V100 GPU 和海光 Z100/K100 DCU 等异构平台上，对 DeepGen 进行了系统性的实验验证。实验结果表明，在矩阵乘法测试中，DeepGen 相较于 cuBLAS 和 rocBLAS 分别实现了最高 1.26 倍和 1.24 倍的性能提升；在注意力机制测试中，相较于 Triton，性能提升最高达 3.71 倍；在端到端推理实验中，DeepGen 在 BERT、GPT-2 和 Llama-2 等模型上均实现了稳定的性能提升，展现出卓越的工程实用性和跨平台适应性。此外，数值精度验证结果表明，DeepGen 生成的算子输出与 PyTorch 基准测试结果高度一致，这表明本文提出的方法既能提升性能，又能确保强健的数值正确性和工程可靠性。
Transformer 架构在自然语言处理、代码生成及大模型推理等任务中得到了广泛使用，矩阵乘法（Matrix Multiplication, Matmul）和多头注意力机制（Multi-Head Attention，MHA）作为其关键算子，它们执行效率已经成为影响模型部署性能的主要因素。异构硬件平台在模型执行、内存层次结构、软件生态系统上的差异比较大，使用高性能库或手动编写优化内核的传统方法，会面临适应性复杂、开发成本高、性能可移植性不够等问题。同时，现在的深度学习编译器在块级逻辑表达、低级内存访问控制和跨平台性能可移植性上还有局限，不容易达到 Transformer 关键算子高性能自动代码生成的目的。针对上述问题本文提出 DeepGen--一个基于 MLIR 的高性能算子生成框架，解决 Transformer 模型关键算子遇到的以上问题。本文的主要贡献是：

(1) 本文提出了一种的线程块级和线程级的双层中间表示（IR）方法。为了解决现有编译器面临的挑战，也就是在代码生成过程中难以保留块级语义、无法显式表达计算和内存访问结构，我们设计了基于 MLIR（Multi-Level Intermediate Representation）的 DeepGen 方言（Dialect）。该方言对矩阵乘法、多头注意力机制里的线程块级别计算逻辑、数据访存还有并行映射关系给出了一个统一模型。该方法通过显式定义 Alloc Buffer、Copy、Matmul 和 Block 等操作（Operation），把计算逻辑、数据访存进行解耦，这样做也为跨平台性能移植和后续优化提供了统一且清楚的表示基础。

(2) 本文设计了一种根据推断 IR 降级（Lowering）信息的自动降级方式。这是一种关注数据依赖关系的索引推断方法，用来处理线程块级 IR 向低级线程级 IR 转换中线程级索引计算的问题。它依据算子的计算方式以及算子间的缓冲区（Buffer）依赖关系，自行计算线程块和线程束内部的索引布局，最后支持 IR 的降级转换。该方法能够使算子语义在不一样的抽象层级里正确传递，给之后的内存访问优化、并行调度和底层代码生成打下了基础。

(3) 本文提出了一套针对异构平台的跨平台优化 Pass 集合以及自动调优方法。优化 Pass 主要采用设计软件流水线（Software Pipeline）、局部规约轴切分（Local Split K）、线程块访存重排（Block Swizzle）和离散化访存（Access Scatter）等技术，来对不同硬件平台进行优化。优化 Pass 在计算调度、规约拆分、线程块映射、共享存储访问等多个方面进行了处理，进而提高所生成代码的执行效率。其次为解决大规模参数空间和手动调优成本高的问题，本文构建了一个依靠空间剪枝的自动调优框架。该框架依靠生成候选参数空间和剔除无效配置，自动进行面向目标硬件的优化执行配置搜索，从而使算子生成的自动化程度得到提高。

(4) 本文在 NVIDIA A100 和 V100 GPU 与海光 Z100 和 K100 DCU 的异构平台上，对 DeepGen 进行了实验验证。实验结果表明，在进行矩阵乘法测试时，DeepGen 比 cuBLAS 和 rocBLAS 分别达到了最高 1.26 倍和 1.24 倍的性能提升。在注意力机制测试里，相较 Triton 在 NVIDIA 和海光平台上的性能提高最高分别达到 4.12 倍和 4.87 倍。端到端推理实验结果说明，DeepGen 在模型 BERT、GPT-2 和 Llama-2 上都明显提高性能，具有不错的工程应用价值、跨平台适应能力。数值精度验证结果说明，DeepGen 生成的算子输出和 PyTorch 基准测试结果非常一致，表明本文提出的方法可以提高性能，并且能够保证数值正确性和工程可靠性。
}

\eabstract{
% Transformer has been widely adopted in tasks such as natural language processing, code generation, and large-model inference. As its key operators, matrix multiplication (Matmul) and multi-head attention (MHA) have become major factors affecting deployment performance. However, heterogeneous hardware platforms differ significantly in execution model, memory hierarchy, and software ecosystem. As a result, traditional approaches based on vendor-specific high-performance libraries or manually optimized kernels often suffer from poor adaptability, high development cost, and limited performance portability. Meanwhile, existing deep learning compilers still have limitations in block-level logic representation, low-level memory access control, and cross-platform performance portability, making it difficult to achieve high-performance automatic code generation for Transformer operators. To address these issues, this thesis proposes DeepGen, an MLIR-based high-performance operator generation framework for key Transformer operators. The main contributions of this work are as follows.

% (1) This thesis proposes a dual-level intermediate representation (IR) method at the thread-block level and thread level. To address the challenges faced by existing compilers—namely, the difficulty of preserving block-level semantics during code generation and the inability to explicitly represent computation and memory access structures—we design the DeepGen dialect based on MLIR (Multi-Level Intermediate Representation). This dialect provides a unified model for thread-block-level computation logic, data movement, and parallel mapping in matrix multiplication and multi-head attention. By explicitly defining operations such as Alloc Buffer, Copy, Matmul, and Block, this method decouples computation logic from data movement, thereby providing a unified and clear representation foundation for cross-platform performance portability and subsequent optimizations.

% (2) This thesis designs an automatic lowering method based on inferred IR lowering information. Specifically, it is an index inference method that focuses on data dependencies and is used to handle thread-level index computation during the transformation from thread-block-level IR to lower-level thread IR. Based on the computation pattern of operators and the buffer dependencies among them, it automatically derives index layouts within thread blocks and warps, and finally supports the lowering transformation of IR. This method ensures that operator semantics are correctly propagated across different abstraction levels, laying the foundation for subsequent memory access optimization, parallel scheduling, and low-level code generation.

% (3) This thesis proposes a set of cross-platform optimization passes and an auto-tuning method for heterogeneous platforms. The optimization passes mainly include techniques such as Software Pipeline, Local Split-K, Block Swizzle, and Access Scatter, which are used to optimize code generation for different hardware platforms. These passes improve execution efficiency from multiple aspects, including computation scheduling, reduction decomposition, thread-block mapping, and shared-memory access. In addition, to address the challenges of large parameter spaces and the high cost of manual tuning, this thesis constructs an auto-tuning framework based on search-space pruning. By generating candidate parameter spaces and removing invalid configurations, the framework automatically searches for optimized execution configurations for target hardware, thereby improving the level of automation in operator generation.

% (4) This thesis evaluates DeepGen on heterogeneous platforms including NVIDIA A100 and V100 GPUs, as well as Hygon Z100 and K100 DCUs. Experimental results show that, in matrix multiplication tests, DeepGen achieves up to 1.26× and 1.24× performance improvement over cuBLAS and rocBLAS, respectively. In attention mechanism tests, it achieves up to 4.12× speedup over Triton. End-to-end inference experiments further show that DeepGen delivers clear performance gains on BERT, GPT-2, and Llama-2, demonstrating strong engineering value and cross-platform adaptability. Numerical accuracy validation also shows that the outputs of DeepGen-generated operators are highly consistent with PyTorch baselines, indicating that the proposed method can improve performance while maintaining numerical correctness and engineering reliability.

Transformers are now used in many fields, such as processing natural language, generating code, and large model inference. In Transformer, the two operations of matrix multiplication (Matmul) and long-head attention (MHA) are particularly important, and they directly affect the performance of the model in actual use. But now there is a problem, that is, different hardware platforms are too different, such as execution model, memory structure and software ecology are not the same. In the past, we used to use the high-performance libraries provided by manufacturers or manually optimized the kernel, but these methods are not very adaptable, the development cost is high, and the performance of other platforms may become worse. Existing deep learning compilers also have their own problems, such as not doing enough in block-level logical representations and underlying memory access control, making it difficult to automatically generate high-performance Transformer operation code. To solve these problems, we have created a new framework called DeepGen, which is based on MLIR and is designed to generate high-performance Transformer operation code. The main contributions of this work are as follows.

First, we propose a two-level intermediate representation method, including thread block level and thread level IR. Because existing compilers have difficulty preserving block-level semantics when code generation, and cannot clearly represent compute and memory access structures, we designed the DeepGen dialect based on MLIR. This dialect unifies the parallel mapping of threaded block-level computation, data movement, and matrix multiplication and multi-head attention. We defined operations such as Alloc Buffer, Copy, Matmul, and Block, which separate computational logic and data movement, laying the foundation for cross-platform performance porting and subsequent optimization. 

The second thing is that we designed an automatic reduction method that focuses primarily on data dependence and handles thread-level indexing calculations through indexing inference. Specifically, according to the operator's calculation mode and buffer dependence, the index layout in the thread block and warp is automatically derived, and finally the conversion of IR is supported. This ensures that the operatic semantics are passed correctly at different levels of abstraction, preparing for later memory access optimizations and code generation. 

Third, we have developed a cross-platform optimization method, including Software Pipeline, Local Split K these technologies to optimize code generation for different hardware platforms. These methods improve execution efficiency in terms of computational scheduling, thread block mapping, and more. In addition, considering that manual tuning is too laborious, we also made an automatic tuning framework, which will become a candidate parameter space, and then remove those useless configurations and automatically find the execution scheme that is most suitable for the target hardware, so that the code generation is more automated. 

Finally, we tested DeepGen on NVIDIA A100, V100 GPUs, and Hygon Z100, K100, and other DCUs. The results of the test showed that DeepGen was much faster than cuBLAS and rocBLAS in matrix multiplication, with a maximum performance of 1.26× and 1.24×. In the attention mechanism evaluation, the maximum performance improvement over Triton reached 4.12× on the NVIDIA platform and 4.87× on the Hygon platform. We also did end-to-end inference tests on BERT and GPT-2 models and found that DeepGen did deliver significant performance improvements. Accuracy testing also showed that the code generated by DeepGen is basically the same as the PyTorch baseline results, indicating that our approach improves performance while also ensuring numerical accuracy and engineering reliability.
}
