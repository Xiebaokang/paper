\chapter{总结与展望}

% 第一节
\section{总结}
% 随着 Transformer 模型在自然语言处理和大模型推理场景中的广泛应用，矩阵乘法与多头注意力机制等关键算子的高效实现已成为影响模型部署性能的核心因素。与此同时，异构硬件平台的快速发展使得传统依赖人工优化内核和厂商专用高性能库的实现方式面临较高的开发、移植与维护成本。针对现有深度学习编译框架在块级逻辑表达、跨平台优化和性能迁移方面的不足，本文围绕“基于 MLIR 的 Transformer 模型关键算子自动代码生成”这一主题，设计并实现了面向异构平台的高性能算子生成框架 DeepGen，并对其关键机制与实验效果进行了系统研究。

% 本文的主要工作可概括为以下几个方面。

% 首先，本文针对现有编译框架在高性能算子生成中存在的表示能力不足问题，提出了一种面向 Transformer 关键算子的块级分层中间表示方法。基于 MLIR 的多级表示机制，本文设计了 DeepGen Dialect，用于显式刻画矩阵乘法与注意力机制中的块级计算逻辑、数据搬运行为和并行结构，从而弥补了高层张量表示与底层硬件执行之间的语义空白。相较于传统依赖循环展开或隐式布局推导的方式，该方法能够更直接地表达多级分块、线程协作和共享存储访问等关键信息，为后续优化与降级提供了统一而清晰的表示基础。

% 其次，围绕块级 IR 向线程级实现的映射问题，本文设计了基于 Lowering 信息推断的降级机制。通过对缓冲区布局、线程映射关系和索引规则的结构化推导，实现了从高层块级算子描述到底层线程执行逻辑的渐进式转换。该机制不仅保证了算子语义在不同抽象层级间的正确传递，也为后续访存优化、并行调度和代码生成奠定了基础，使得编译过程能够兼顾表达完整性与实现效率。

% 再次，针对异构硬件平台在执行模型和存储层次上的差异，本文提出了一套通用的跨平台优化 Pass 集合，包括 Software Pipeline、Local Split-K、Block Swizzle 和 Access Scatter 等关键优化技术。这些优化从计算调度、规约拆分、线程块映射和共享存储访问等多个层面提升了生成代码的执行效率。尤其是通过将“计算逻辑”与“数据搬运”显式解耦，DeepGen 能够针对不同硬件平台重写数据拷贝与访存索引，从而更有效地控制 Bank 冲突与缓存利用，增强了框架对异构硬件的适配能力。

% 此外，针对高性能算子生成中普遍存在的调优空间巨大、搜索成本高的问题，本文还设计了基于空间剪枝的自动调优机制。该方法以线程块和线程负载分块大小等参数为核心，构建结构化搜索空间，并结合性能测试缓存与参数遍历策略，从而自动获得较优的算子配置。该机制降低了人工调参负担，使 DeepGen 具备了更强的自动化生成能力。

% 在实验验证方面，本文从算子级性能、数值精度和端到端推理性能三个层面对 DeepGen 进行了系统评估。实验结果表明，在 NVIDIA A100/V100 GPU 以及海光 Z100/K100 DCU 等异构平台上，DeepGen 均表现出较好的性能与可移植性。其中，在矩阵乘法测试中，DeepGen 相比 cuBLAS 和 rocBLAS 分别实现了平均 1.10 倍和 1.18 倍的性能提升；在注意力机制测试中，相较 Triton 取得了显著加速，平均可达 2.93 倍；在端到端推理任务中，对 BERT、GPT2 和 LLaMA2 等模型也实现了稳定的性能收益。同时，精度验证结果表明，DeepGen 生成算子的输出与 PyTorch 基准结果高度一致，说明本文提出的方法在提升性能的同时保持了良好的数值正确性。

% 综上所述，本文围绕 Transformer 模型关键算子的自动代码生成问题，提出了基于 MLIR 的 DeepGen 框架，并在中间表示设计、降级推断、跨平台优化和自动调优等方面开展了系统研究。实验结果验证了该框架在异构硬件环境下的有效性、准确性与可移植性。本文的研究工作为 Transformer 模型关键算子的高性能自动生成提供了一种可行思路，也为后续面向更复杂硬件和更大规模模型的编译优化研究奠定了基础。

Transformer 模型在自然语言处理、大模型推理中的使用范围已经扩大，让矩阵乘法和多头注意力等主要算子的实现效率变得重要，这会影响到模型部署的性能。异构硬件平台的发展加快了速度，传统依靠人工优化内核和厂商专用库的实现方式不得不承担较高的开发与移植，并且维护成本也变得比较大。现在深度学习编译框架在块级逻辑表达、跨平台优化、性能迁移方面有不足。本文运用 MLIR，解决 Transformer 模型主要算子的自动代码生成问题，并且设计及实现了面向异构平台的高性能算子生成模式 DeepGen。

本文的主要内容主要包括下面这些方面。本文提出面向 Transformer 主要算子的块级分层中间表示方式，以解决现有编译框架在生成高性能算子时表达能力不足的问题。本文凭借 MLIR 的多级表示机制设计了 DeepGen Dialect 用来对矩阵乘法、注意力机制里的块级计算逻辑、数据搬运行为和并行结构进行显式描述，这给后续优化和降级提供了统一的表示基础。

针对块级中间表示向线程级实现映射的问题，本文设计了依靠降级信息（LowerInfo）进行推断的转换方式。该方式对缓冲区布局、线程映射关系和索引规则的结构进行推导，以实现从高层块级描述到底层线程执行逻辑的渐进式转换，并且可以支持访存优化、并行调度和代码生成。

针对异构硬件平台在执行模型和存储层次上的差异，本文提出了一套通用的跨平台优化方案，包括 Software Pipeline、Local Split K、Block Swizzle 和 Access Scatter 等技术。这些方法通过计算调度、规约拆分、线程块映射和共享存储访问等操作，使生成软件代码的执行效率得以提高，同时也改善了该模式对各类适配硬件平台的能力。

针对调优空间较大和搜索成本较高的问题，本文设计了通过空间剪枝实现自动调优的方式。该方式依靠线程块、线程负载分块等参数来构成搜索空间，并且利用性能测试缓存和参数遍历策略，自动得到较优的配置，这样可以减少人工进行参数调整的工作量。

实验结果说明，DeepGen 在 NVIDIA GPU 和海光 DCU 平台上都拥有不错的性能和可移植性。在矩阵乘法测试中，相比 cuBLAS 和 rocBLAS，本文实现平均 1.04 倍和 1.03 倍性能提高。在注意力机制测试中，相较于 Triton 在 NVIDIA 和海光平台上平均加速分别可以达到 2.55 倍和 3.09 倍。在端到端推理中，本文对 BERT、GPT2 和 LLaMA2 等模型进行实验，同样获得了稳定的收益。精度测试的结果表明，生成算子的输出结果表示和PyTorch基准结果保持了高度的一致性。

\section{展望}
% 尽管本文围绕 Transformer 模型关键算子的自动代码生成问题，设计并实现了基于 MLIR 的 DeepGen 框架，并在多项任务中取得了显著的实验效果，但仍有若干方向值得进一步研究。

% 首先，应深化对新型异步硬件单元的支持。当前 DeepGen 主要面向 FP32 精度下的 CUDA Core 执行路径，尚未充分启用 Tensor Core 等专用硬件加速特性。随着硬件体系结构的演进，TMA、WGMMA 等异步执行机制已成为实现高性能计算的关键。相较于传统同步模式，异步执行单元能显著提升数据传输与计算的并行度，为算子优化提供更大空间。未来可在 DeepGen 现有分层 IR 的基础上，扩展对异步拷贝、异步矩阵乘及多阶段流水执行语义的表示能力，并设计相应的 Lowering 策略与优化 Pass，使框架逐步覆盖混合精度与异步执行场景，提升生成代码的性能上限。

% 其次，需探索更高效的启发式搜索与调优策略。虽然 DeepGen 已通过结构化参数空间定义与分层剪枝显著缩小了搜索范围，但本质上仍以遍历测试为主。当参数维度持续扩展、硬件特征更趋复杂时，单纯依赖规则剪枝难以兼顾搜索效率与最优性。未来有必要在现有框架中引入启发式搜索算法，例如基于经验模型的候选排序、基于历史结果的增量搜索、或结合性能预测模型的学习型调优方法。通过将启发式搜索与现有剪枝机制相结合，有望在保证搜索质量的同时大幅减少无效编译与测试次数，增强 DeepGen 对大规模任务与复杂硬件平台的适应能力。

% 此外，应推动算子级优化向图级协同优化演进。随着模型规模的扩大，高性能代码生成不仅需关注单个算子的局部最优，更需考虑大范围内的图级协同。未来可进一步探索跨算子融合、运行时调度协同以及面向整图推理的统一优化策略，使算子级代码生成能力与图级编译优化紧密结合，从而在真实的大模型部署场景中释放更大的系统级性能潜力。

% 综上所述，未来 DeepGen 的发展将围绕异步硬件支持、高效调优策略及图级优化展开，不断提升其在异构平台上的性能生成能力与工程实用价值。
尽管本文针对 Transformer 模型中主要算子的自动代码生成问题，设计并实现了使用 MLIR 的 DeepGen 模式，并且达到了比较理想的实验效果，但是仍然存在进一步优化的余地。

DeepGen 目前主要针对 FP32 精度上的 CUDA Core 执行路径，对 Tensor Core、TMA、WGMMA 等异步硬件机制的支持还不够充分。由于硬件架构在不断演进，异步执行已成为提高计算和数据搬运重叠度的主要方式。后续工作可以在现有分层中间表示的基础上，增强对异步拷贝、异步矩阵乘及多阶段流水执行的表示能力，并且设计相应的降低策略与优化遍，从而提高该模式在混合精度和异步执行场景下的性能上限。

在自动调优方面，目前 DeepGen 主要还是使用参数遍历和规则剪枝方法。参数空间如果一直扩大，硬件特征变得更加复杂，该方式在效率和搜索质量上会表现出不足。后续本文可以将启发式搜索、历史结果驱动的增量搜索、性能预测模型等方法进行引入，以便减少无效编译和测试的次数，从而提高调优效率。

未来还可以把优化范围从算子级扩展到图级，分析跨算子融合、运行时调度共同优化和整的所有图推理优化，让代码生成能力、图级编译优化之间进行结合变得更加紧密。

总体来看，DeepGen 后续仍可围绕异步硬件支持、高效调优和图级优化三个方向继续完善，以进一步提升其在异构平台上的性能与实用性。
