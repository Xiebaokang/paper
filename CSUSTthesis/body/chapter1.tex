\chapter{绪论}
% 第一节
\section{研究背景及意义}
近年来，以 Transformer~\cite{vaswani2017attention}架构为核心的大语言模型（Large Language Models, LLMs）在自然语言处理、计算机视觉及多模态学习等领域取得了突破性进展。从早期的 BERT、GPT 系列到如今的 LLaMA、混合专家模型（Mixture of Experts, MoE），模型参数量与上下文长度不断攀升，使得深度学习任务的计算需求呈现爆炸式增长。在模型的推理部署阶段，性能表现直接关系到实时交互体验与服务成本。研究表明，Transformer 模型的核心计算负载主要集中在矩阵乘（Matrix Multiplication, Matmul）以及多头注意力机制（Multi-Head Attention, MHA）中。尽管矩阵乘在算力密集的硬件上已有较成熟的优化，但 MHA 由于涉及多个算子的链式组合且具有高度复杂的访存模式，已成为制约大模型推理延迟与吞吐量的核心瓶颈。

为了加速注意力机制，业界涌现了如 FlashAttention、Memory-Efficient Attention 等优秀的底层优化方案，通过算子融合（Operator Fusion）与分块计算（Tiling）显著提升了访存效率。然而，当前的加速成果高度依赖于 NVIDIA 的 CUDA 生态及其专有的硬件特性（如 Tensor Core、异步拷贝指令等）。与此同时，国内外异构计算加速设备层出不穷，尤其是以海光 DCU、寒武纪 NPU 等为代表的国产芯片正在快速崛起。由于不同硬件架构的指令集、存储层次及线程调度模型存在显著差异，现有厂商提供的底层高性能库（如 cuBLAS、rocBLAS）通常呈现“烟囱式”开发模式。开发者往往需要针对特定平台投入巨大的精力进行手工代码编写与性能调优，这种高度绑定的开发模式导致了严重的软件生态碎片化，极大地提高了算法跨平台迁移的成本与维护难度。

尽管自动化编译技术被视为解决上述问题的关键路径，但现有的深度学习编译器在面对 Transformer 类算子与异构硬件的深度适配时，仍面临着“语义鸿沟”与“控制力不足”的双重挑战。一方面，以 TVM 为代表的传统编译器采用基于循环（Loop-based）的低级中间表示，在代码降级（Lowering）过程中过早地丢弃了张量计算的块级语义，导致编译器难以有效利用Tensor Core等专用硬件加速单元。另一方面，以 Triton 为代表的新兴编译器虽然引入了块级抽象，但其严重依赖隐式的布局推断与黑盒化的数据搬运机制。在面对海光 DCU 等具有特殊存储结构（如特定的 Bank 冲突模式）的国产硬件时，这种隐式机制往往无法生成最优的访存指令，且开发者难以通过显式手段干预数据通路以消除硬件瓶颈。因此，探索一种既能保留高层块级语义，又能显式解耦计算与数据搬运、从而实现精细化硬件控制的算子生成新范式，成为了当前亟待解决的关键问题。

面对硬件异构化、软件生态碎片化以及现有编译技术局限性的多重挑战，研究高效的跨平台算子自动代码生成技术具有重大的科学意义与应用价值：
\begin{enumerate}
\item 突破性能可移植性瓶颈：针对传统编译器在块级逻辑表达及底层访存控制上的不足，利用 MLIR（Multi-Level Intermediate Representation）的多级抽象与方言（Dialect）机制，构建一套解耦计算逻辑与数据搬运的编译框架。通过引入显式的块级中间表示（Block-level IR），在保留高层算子语义完整性的同时实现对底层硬件特性的精确抽象与直接映射，从而在跨平台可移植性层面显著降低了异构硬件适配与拓展的工程复杂度。
\item 降低国产硬件适配门槛：针对软件生态相对薄弱的国产异构设备，自动代码生成技术可以替代高强度的手工调优工作。利用通用编译优化 Pass（如局部规约轴并行、离散访存优化等），可以自动处理不同硬件架构下的访存冲突与流水线编排，快速生成匹配硬件特性的高性能内核，助力国产算子库与软件生态的快速补齐。
\item 提升大模型部署效率：结合自动调优（Auto-tuning）技术，该研究能够针对不同硬件参数（如寄存器容量、共享内存带宽）构建参数搜索空间，并利用剪枝策略自动搜索最优执行配置。这使得模型在推理过程中，无论是在成熟的 NVIDIA GPU 还是新兴的国产加速卡上，均能保持接近硬件峰值的算力利用率，显著降低大模型的部署成本。
\end{enumerate}

综上所述，本文提出的基于 MLIR 的 Transformer 模型关键算子自动代码生成研究，不仅能够为大模型的高效部署提供坚实的技术底座，更能为异构计算平台的生态构建提供一种低成本、高效率且具备理论深度的通用技术方案。

\section{国内外研究现状}

\subsection{Transformer 模型及其关键算子优化演进}
自 Transformer 架构提出以来，自然语言处理领域迎来了飞跃式的发展。为了追求更优异的模型性能与更强大的泛化能力，学术界与工业界长期遵循扩展定律（Scaling Law）。该定律指出，通过持续堆叠 Transformer 层数、增加隐藏层维度以及扩大注意力头数，可以显著提升模型捕捉复杂语义的能力。如图~\ref{1_1}~展示了Transformer 模型参数量随时间变化的趋势，从早期的 BERT-Base 模型发展至 GPT-3，再到后来万亿参数级别的混合专家模型，模型参数量经历了从亿级到千亿级甚至万亿级的指数级跨越，从而在逻辑推理、文本生成及代码补全等任务上取得了质的突破。

然而，随着模型规模的不断攀升，训练与推理成本的急剧上涨已成为限制大模型大规模商业化落地的主要障碍。针对这一挑战，近期以 DeepSeek V3 和 R1 为代表的新一代模型探索出了低成本兼顾高性能的新路径。该类模型不再单纯依赖盲目堆叠参数，而是通过高效的混合专家 MoE 架构创新，并引入强化学习（Reinforcement Learning, RL）驱动的后训练机制，大幅增强了模型的推理效能。这种范式的转变虽然在算法层面降低了智能获取的边际成本，却对底层推理系统提出了更为苛刻的要求：无论是复杂的稀疏激活模式，还是对超长上下文窗口的支持，都使得在有限硬件资源下实现低延迟、高吞吐的推理成为核心挑战。因此，对底层推理引擎的极致优化已成为实现大模型技术普惠的关键环节。

在 Transformer 模型的推理过程中，计算负载主要集中在矩阵乘法与多头注意力机制这两个核心算子上。如图~\ref{1_2}~所示的 Transformer 模型推理时间开销分析表明，虽然通用矩阵乘 GEMM 占据了大量计算时间，但随着序列长度的增加，多头注意力机制 MHA 的计算复杂度与显存占用呈二次方增长趋势。这使得 MHA 逐渐取代矩阵乘法，成为制约长文本推理性能的首要瓶颈。

传统的 MHA 实现方式通常遵循“计算与访存分离”的模式，即首先计算查询向量与键向量的点积以得到注意力分数矩阵，将其写入高带宽全局显存（HBM），随后读取该矩阵进行 Softmax 归一化操作，最后再次写入显存。这种频繁的显存读写操作引发了严重的内存墙（Memory Wall）问题，导致 GPU 的计算单元因长期等待数据而处于闲置状态，实际吞吐量远低于硬件理论峰值。
为了解决这一问题，FlashAttention 系列算法应运而生，通过分块计算与重计算策略彻底重塑了 MHA 的执行范式。第一代 FlashAttention 利用 GPU 片上高速共享内存（SRAM）作为缓存，通过算子融合技术将矩阵乘、Mask 操作、Softmax 以及 Dropout 等多个步骤在片上一次性完成，避免了中间大矩阵对全局显存的反复读写，成功将访存复杂度从二次方降低为线性。随后的 FlashAttention-v2 进一步优化了线程块层面的任务划分与并行策略，大幅减少了非矩阵乘操作带来的同步开销，并通过优化线程束级的数据布局有效缓解了共享内存的 Bank 冲突，显著提升了在 NVIDIA Ampere 架构上的资源占用率。最新的 FlashAttention-v3 则针对 NVIDIA Hopper 架构特性进行了深度定制，通过利用 WGMMA 矩阵指令集与张量存储加速器（Tensor Memory Accelerator, TMA）等硬件特性，实现了计算流水线与访存流水线的深度重叠，进一步逼近硬件性能极限。

然而，这些顶尖的优化成果大多深度绑定 NVIDIA CUDA 指令集及其底层的计算资源分配模式。一旦部署环境切换至海光 DCU 或寒武纪 NPU 等国产异构平台，由于缺乏等效的高性能内核支持，模型性能往往会出现断崖式下跌。这种对特定硬件生态的深度依赖，使得 Transformer 模型的跨平台部署面临着巨大的代码重构成本与维护压力。

综上所述，虽然以 FlashAttention 为代表的手工深度优化算子极大地推高了 Transformer 模型的推理性能天花板，但其高昂的迁移成本与封闭的生态特性，已难以适应当前异构计算硬件百花齐放的行业现状。在硬件架构快速迭代与模型算子日益复杂的双重压力下，单纯依赖人工针对特定硬件编写高性能内核的传统模式已难以为继。如何摆脱对单一硬件厂商底层库的过度依赖，在保证算子极致性能的同时实现跨平台的自动化部署，成为了学术界与工业界共同探索的焦点。这也促使研究者将目光转向深度学习编译器技术，试图通过自动化的代码生成与优化路径来打破这一僵局。


\subsection{现有深度学习编译器的技术路径与局限}
随着深度学习模型复杂度的提升，仅仅依赖手工优化内核已难以应对算子形态的快速演进与硬件后端的碎片化问题。为此，学术界与工业界发展出了一系列深度学习编译器，旨在通过中间表示的构建与变换，实现从高层计算图到底层机器码的自动化生成。根据中间表示抽象层次的不同，现有的编译器技术主要可以分为两大流派：一是以 XLA、TVM 为代表的基于标量循环（Loop-based）的编译器，二是以 Triton 为代表的基于块级抽象（Block-based）的领域特定编译器。如图~\ref{1_3}~所示，展示了TVM、Triton和我们们工作在GEMM算子上的中间表达代码对比，同时表~\ref{tab:table1}展示了我们工作与TVM及Triton类型的编译器的核心设计差异。尽管它们在特定时期推动了技术发展，但在面对 Transformer 架构在异构平台上的极致性能需求时，均表现出了不同程度的局限性。

在以 XLA、Halide 和 TVM 为代表的第一代深度学习编译器，确立了计算与调度分离的核心设计理念。这类编译器的核心思想是将高维张量运算Lowering为最基础的标量元素级读写操作，并用多层嵌套循环来描述计算逻辑。如图~\ref{1_3}~(a)，TVM 采用张量表达式（Tensor Expression, TE）作为前端输入，用户或自动调度器可以通过调度原语对循环结构进行变换。这些变换包括循环分块以适应 Cache 大小、循环重排以优化数据局部性、以及向量化和循环展开等。为了解决庞大调度空间中的参数选择难题，AutoTVM 和 Ansor 引入了机器学习技术，通过模板匹配或草图生成的方式，结合 XGBoost 等算法自动搜索最优的循环参数配置，从而在 CPU 和早期 GPU 上实现了可与手工库媲美的性能。

尽管这种基于循环的表达方式具有极高的通用性，但它导致了严重的“语义丢失”问题。当编译器将矩阵乘法拆解为 $for (i, j, k)$ 的标量循环后，原本的矩阵计算语义被破坏。在面向现代 GPU 架构时，硬件厂商为了提升算力，引入了专用的矩阵加速单元（如 NVIDIA 的 Tensor Core、AMD 的 Matrix Core）。这些单元通常要求数据以特定的分块形状和布局加载。为了利用这些硬件特性，基于循环的编译器必须执行复杂的张量化（Tensorization）过程，即通过模式匹配（Pattern Matching）尝试从破碎的标量循环中重建出矩阵语义，再映射到硬件内建指令。这一过程极其脆弱，往往因为循环边界的微小变化或复杂的算子融合逻辑而匹配失败，导致生成的代码回退到低效的标量计算模式，无法释放硬件的峰值算力。

为了解决标量循环无法自然映射到矩阵加速单元的问题，以 OpenAI 推出的 Triton 为代表的新一代领域特定语言（Domain Specific Language, DSL）应运而生，并迅速成为 PyTorch 2.0 的默认后端技术。如图~\ref{1_3}~(b)，Triton 摒弃了细粒度的标量循环描述，转而采用块级的编程模型。在 Triton 中，基本的操作单元不再是单个浮点数，而是参数化的块（tile）。开发者使用 tl.load、tl.dot、tl.store 等原语直接对张量块进行操作。这种设计使得编译器能够天然地保留矩阵运算的语义，从而更容易地映射到 Tensor Core 等块级硬件指令上。此外，Triton 采用类似 Python 的语法，大幅降低了高性能算子的开发门槛，使得算法工程师能够以较高的抽象层次编写 GPU 内核。

尽管 Triton 在 NVIDIA GPU 上表现优异，但其设计哲学中包含的大量隐式推断机制，在面对跨平台异构部署时成为了新的阻碍。在数据布局与访存方面，Triton 倾向于自动管理共享内存的数据布局（Layout）和存取模式（Swizzle）。虽然它提供了 convert\_layout 等接口，但具体的内存排布逻辑对开发者是半透明的“黑盒”。在 NVIDIA 架构上，这套启发式策略运行良好，但是当迁移至海光 DCU 或其他国产 AI 芯片时，由于这些硬件具有不同的 Cache Line 大小、Bank 数量及冲突模式，Triton 的隐式策略往往无法生成最优的无冲突访问模式。开发者难以像在 CUDA 或汇编中那样，通过位运算显式地控制每一个数据的物理存储位置，导致性能在异构平台上出现严重衰退。
而在调度策略方面，Triton 的许多编译器优化 Pass 是针对 NVIDIA 硬件特性深度定制的。在移植到其他架构时，往往需要重写大量后端逻辑。这种架构特定优化与语言通用性之间的矛盾，限制了其在广泛异构算力环境中的普及。

综上所述，现有编译方案在适配Transformer大模型与异构硬件时，面临三大核心挑战：其一，表达抽象与硬件控制间的断层——通用循环抽象难以有效利用张量核心，而高层块级DSL又缺乏对内存层次结构的显式控制。其二，性能可移植性不足——现有体系严重依赖CUDA生态，导致在多样化的硬件后端难以实现统一的高性能。其三，自动化调优效率低下——随着算子融合等优化加深，调度搜索空间呈组合爆炸，缺乏领域知识引导的搜索难以收敛至高效解。针对这些挑战，本研究旨在构建一个兼具表达力、可移植性与高效搜索能力的编译框架。

\subsection{硬件厂商高性能库的依赖与挑战}
在如 PyTorch、TensorFlow 等深度学习框架的底层实现中，硬件厂商提供的高性能算子库长期扮演着基石的角色。为了在激烈的硬件竞赛中通过基准测试，GPU 厂商以及国产芯片厂商均投入了庞大的工程团队，针对特定的指令集架构和微架构特性，手工编写并调优核心数学库。这些库以 NVIDIA 的 cuBLAS、cuDNN 以及 AMD 的 rocBLAS、MIOpen 以及 Tensile 为代表，代表了当前通用矩阵乘与卷积运算的性能“天花板”。其高效性源于对底层硬件的极致压榨：一方面，库开发者通过汇编级微调，直接使用底层汇编语言编写内核，以精确控制寄存器分配、指令流水线排布以及指令级并行。另一方面，库中包含成百上千个针对不同代际 GPU 架构和特定矩阵尺寸优化的内核变体，运行时系统会根据输入形状通过启发式算法动态选择最优实现。因此，在标准的矩阵乘法测试中，任何自动生成的代码想要超越这些经过数十年积累的手工库都极具挑战。

尽管厂商库在单一算子的性能上表现卓越，但在大模型快速迭代与异构部署的现实场景下，其“黑盒”与“硬编码”特性暴露出了严重的局限性。首先，不同硬件厂商的算子库存在严重的生态隔离，形成了“烟囱式”的开发模式。这种隔离不仅体现在 NVIDIA 使用 CUDA 生态而海光 DCU 使用基于 ROCm 的 HIP 生态的底层差异上，更表现为 API 不兼容、函数签名和参数定义存在差异等具体问题，导致将模型从 NVIDIA 平台迁移至国产硬件时需要重写大量胶水代码，造成高昂的移植成本和对特定厂商生态的深度依赖。其次，厂商库面临算子融合能力的严重瓶颈，如图~\ref{1_4}~(a)所示为采用厂商提供的单算子实现的MHA和融合MHA的执行过程，其中绿色和蓝色箭头分别表示从HBM中读取和写入操作。在 Transformer 结构中，计算密集型算子与访存密集型算子的交替出现使得中间结果必须反复写回全局显存再重新读取，产生巨大的带宽开销和“内存墙”问题，如图~\ref{1_4}~(b)所示为未融合且反复访存HBM的MHA与融合型只有一次HBM的访存的MHA的时延比较。虽然部分厂商库开始提供有限的融合接口，但这种支持是静态且非通用的，无法灵活适应不断涌现的新型激活函数或复杂的 Attention 变体，导致编译器不得不回退到低效的“内核分裂”执行模式。第三，厂商库的维护周期严重滞后于算法演进的速度。当学术界提出新的低精度格式或稀疏计算模式时，厂商库通常需要数月甚至更久才能提供官方支持，且其优化资源往往集中在头部模型的标准尺寸上，对于大模型中出现的变长序列、非标准形状等“长尾”场景，库函数的启发式选择往往失效，导致性能急剧下降。

综上所述，虽然硬件厂商的高性能库在标准算子上确立了性能标杆，但其生态割裂的现状、算子融合的乏力以及对模型演进的滞后响应，使其难以独自支撑大模型时代对跨平台、端到端极致性能的追求。这迫切呼唤一种能够吸收手工库优化精髓，同时具备编译器灵活性的新型技术方案，这也正是本研究根本出发点。

\section{论文主要研究内容}
针对 Transformer 类大模型在异构硬件部署中面临的适配复杂、手工优化成本高以及现有编译器表现不佳等挑战，本文以实现高性能算子的跨平台自动生成为目标，提出并设计了基于 MLIR 的 DeepGen 编译框架。本文的主要研究内容包括以下四个方面：

1. 面向异构硬件特性的块级分层中间表示（IR）设计与降级机制研究
为了解决传统深度学习编译器在代码降级过程中丢失算子块级语义的问题，本文基于 MLIR 基础设施，设计了一套专用的 DeepGen Dialect。
块级抽象建模：该 IR 体系采用分层设计思想，首先在 Block 级别对矩阵计算进行粗粒度建模，保留了矩阵乘法与注意力机制的高维张量语义。其次，通过显式定义的 Alloc\_Buffer、Copy 与 Block、Matmul 操作，实现了计算逻辑与数据搬运的彻底解耦。这种解耦设计使得编译器能够独立地对访存行为进行精细化控制，为消除异构硬件上的存储访问冲突奠定了基础。
自动化降级推断：针对从 Block 级到 Thread（线程）级的映射难题，本文提出了一种基于数据依赖关系的索引推断算法。该算法能够根据算子的计算模式，自动推导线程块与线程束内的细粒度索引布局，实现了从高层抽象到底层 SIMT（单指令多线程）执行模型的自动化且语义正确的转换，有效屏蔽了底层硬件指令集的差异。

2. 异构平台下通用高性能编译优化 Pass 集合设计
针对 Transformer 关键算子在不同硬件上性能差距，本文设计了一套通用的、可插拔的编译优化 Pass 集合，涵盖了计算调度与访存管理的各个层面。
访存流水线优化：为了掩盖全局显存的高延迟，设计了软件流水线（Software Pipelining）优化 Pass，通过多级缓冲技术实现数据的预取，使计算指令与访存指令在时间上深度重叠。
并行度增强优化：针对 MHA 或者 GEMM 算子中规约维度过大导致线程块并行度不足的问题，提出了局部规约轴拆分（Local Split K） 优化 Pass。该技术将规约维度的计算任务切分至多个逻辑线程组，并在片上共享内存中进行中间结果的高效聚合，显著提升了硬件占用率。
访存冲突消除优化：针对海光 DCU 等硬件架构中特殊的 Bank 冲突问题，提出了离散化访存（Access Scatter） 与线程块重排（Block Swizzle）优化 Pass。通过对访存索引进行数学重写与离散化映射，显式地打散了连续地址的访问模式，在不改变计算语义的前提下彻底消除了共享内存的 Bank 冲突，同时大幅提升了 Cache 的命中率与有效带宽。

3. 基于空间剪枝策略的算子自动调优框架构建
为了解决手动调整编译参数耗时费力且难以收敛至最优解的问题，本文构建了一个闭环的自动性能调优框架。
参数空间生成：该框架首先根据输入算子的形状及线程块级别的分块大小，生成线程块级中间表达，空间参数生成算法会通过分析IR中各个操作（operation）及其之间的依赖关系进行空间的参数的生成，剔除产生冲突和冗余的参数，自动生成包含、线程束数量（Num Warps）、流水线级数（Num Stages）等关键参数的搜索空间。
高效剪枝策略：为了应对参数组合爆炸带来的搜索效率低下的问题，设计了多重剪枝策略。通过静态分析剔除会导致寄存器溢出或资源超限的无效配置，大幅缩小了搜索范围。框架通过实际运行并测量候选配置的性能，最终筛选出适配当前硬件特性的最优参数组合，实现了超越厂商手工库性能的自动化代码生成。

4. 跨平台性能评估与端到端推理验证
为了验证 DeepGen 框架的有效性与泛化能力，本文在 NVIDIA A100 GPU 与 海光 Z100/K100 DCU 两类典型的异构计算平台上进行了广泛的实验。
算子级性能测试：分别对比了 DeepGen 生成的算子与官方高性能库、主流编译器在不同矩阵规模下的吞吐量，并深度分析了两个实验的加速效果及其性能来源。
端到端模型验证：将生成的算子集成至 BERT、GPT-2 及 LLaMA2 等主流 Transformer 模型的推理流程中，验证了优化后的算子对模型整体推理延迟（Latency）的提升效果。
数值精度验证：通过与标准 PyTorch 实现进行对比，计算结果的L2范数的相对误差，证明了本文所提出的 IR 转换与优化策略在保持数值计算正确性方面的可靠性。

\section{本文章节安排}
本文围绕 Transformer 模型关键算子在异构平台上的自动化生成与性能优化这一核心问题展开研究，全文共分为五章，各章节的具体内容安排如下：

第 1 章：绪论。本章首先阐述了以 Transformer 为代表的大语言模型在人工智能领域的爆发式发展及其对底层算力的巨大需求，揭示了当前异构硬件平台（如 NVIDIA GPU 与海光 DCU）在算子开发中面临的软件生态碎片化与适配成本高昂等问题。随后，深入分析了国内外在深度学习编译器及高性能算子库方面的研究现状，重点指出了传统编译器在块级语义表达缺失、异构兼容性差以及新兴 DSL 工具存在的隐式控制局限性。在此基础上，确立了本文的研究动机、主要贡献以及研究内容的整体框架。

第 2 章：相关知识及理论介绍。本章为论文的研究提供了必要的理论基础与背景知识。首先，详细介绍了异构计算加速平台，重点对比分析了 NVIDIA GPU 与海光 DCU 的硬件架构特点及其编程模型。其次，深入解析了 Transformer 模型的网络架构，并对矩阵乘法与多头注意力机制的计算特性及访存瓶颈进行了量化分析。最后，介绍了机器学习编译器的基本原理，重点阐述了 MLIR 框架的设计哲学、Dialect 扩展机制以及代码生成流程，论证了其作为本文技术底座的合理性。

第 3 章：Transformer 模型关键算子的代码生成。本章是本文的核心部分，详细阐述了 DeepGen 编译框架的设计与实现细节。首先，提出了一种面向硬件层次结构的多级 IR 设计，定义了块级与线程级的中间表示，并给出了基于数据依赖关系的 IR 降级推断算法。其次，重点介绍了针对异构硬件特性的通用优化 Pass 集合，详细论述了 Software Pipeline、Local Split K、Block Swizzle以及 Access Scatter等关键优化技术的实现原理。最后，设计了基于空间剪枝的自动调优框架，阐述了参数空间的生成与剪枝策略，以实现最优编译配置的自动化搜索。

第 4 章：实验结果。本章对提出的 DeepGen 框架进行了系统性的实验验证与性能评估。实验依托 NVIDIA A100 GPU 与海光 Z100/K100 DCU 两大异构计算平台，选取了不同规模的矩阵乘法与注意力机制算子作为测试对象。通过与厂商官方高性能库以及主流编译器进行多维度的性能对比，详细分析了 DeepGen 的性能优势。此外，还进行了数值精度验证以及基于 BERT、LLaMA2 等模型的端到端推理性能测试，充分证明了本文方法的有效性与跨平台可移植性。

第 5 章：总结与展望。本章对全文的研究工作进行了总结，概括了本文在异构算子生成、中间表示设计及编译优化机制方面取得的主要成果与创新点。同时，客观分析了当前研究存在的局限性，并结合大模型技术的发展趋势，对未来的研究方向进行了展望。
